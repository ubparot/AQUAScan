22  Stage 2: Pool Testing  The second stage was full-vessel pool testing. This was the first time AquaScan was tested as an assembled system in water, while still keeping the test controlled enough for the vessel to be removed quickly if an issue occurred.  Purpose of the test:   Evaluate AquaScan’s full-system flotation, stability, propulsion, probe deployment, sensor collection, Unity interface, and data output in a controlled water environment.  Testing method:  ●   AquaScan was placed in the pool as a complete assembled vessel to confirm that the full system floated properly and remained stable during operation. This built on the earlier buoyancy checks, but added the full system context of electronics, propulsion, probe deployment, and software use.  ●   Manual control was tested first because the pool was too small for meaningful GPS waypoint navigation. The team used this test to evaluate propulsion performance, including forward movement, turning response, motor output, and overall control in water.  ●   The probe deployment system was tested by lowering and retrieving the probe from the vessel. This helped verify cable payout, retrieval, probe movement, and whether the deployment system could operate while the boat was floating.  ●   Sensor readings were collected from eight sampling points in the pool. These points allowed the team to practice the full data process, including collecting readings, exporting values, organizing the data, and converting the results into graphs.  ●   The Unity operator system was used during the pool test to confirm that AquaScan could display sensor readings, sampling points, and mission data while the vessel operated in water. This helped verify that the physical testing process connected properly to the software, data export, and graphing workflow.  What this stage showed:   Pool testing confirmed that AquaScan could function as a complete integrated prototype. The vessel floated under load, responded to manual control, moved using its propulsion system, deployed and retrieved the probe, collected readings from multiple sampling points, and produced usable data output. The pool test also gave the team a safe environment to fine-tune controls and verify the data workflow before moving to a less controlled outdoor environment.  Stage 3: Pond Field Testing  The third stage was pond field testing, which was the first test where AquaScan was used in an environment that closely matched the project’s intended purpose. Unlike the pool, the pond included natural turbidity, less predictable water clarity, outdoor lighting variation, depth variation, and a larger operating area for realistic water-quality monitoring.  Purpose of the test:   Determine whether AquaScan could operate in a real calm-water environment, collect water-quality readings from multiple locations and depths, and organize those readings into usable mission data.
