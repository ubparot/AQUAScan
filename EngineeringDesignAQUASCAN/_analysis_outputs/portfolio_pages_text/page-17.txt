16  Data Tagging and Measurement Integration  One of AquaScan’s strongest features is that each reading is linked to both location and depth. The GPS module provides the vessel’s position, and the deployment system provides the selected sampling depth. This means the system does not just record a value such as pH or dissolved oxygen. It records:  ●   What was measured  ●   Where it was measured  ●   At what depth it was measured  That combination is what turns the vessel from a simple remote platform into a true scientific discovery tool. Instead of isolated readings, AquaScan produces structured environmental datasets that can be compared spatially and vertically.  Live Visualization in Unity  The Unity interface is also used as the main visualization system during and after operation. In live mode, it can display the boat’s position, movement path, selected sample points, and system status while the mission is taking place. This gives the operator immediate feedback and makes it easier to understand how the vessel is behaving in the field. The interface is also intended to display scientific data visually, allowing users to choose which variable they want to view. For example, the user could select dissolved oxygen, pH, turbidity, or salinity and see those values connected to sampling points on the mission map.  The Unity interface can be used to show:  ●   Boat path and position  ●   GPS sampling points  ●   Live system status and motor outputs  ●   Sensor values associated with each point  ●   Data layers for selected measurements  ●   Mission replay after data collection  Data Output and Analysis  Sensor data is exported in a digital CSV format, which makes it easy to organize, analyze, and reuse. Each record can include measurements such as temperature, pH, dissolved oxygen, turbidity, salinity, GPS position, and sampling depth. Because the data is already structured, it can be imported directly into spreadsheets, graphs, and mapping tools for further analysis.
